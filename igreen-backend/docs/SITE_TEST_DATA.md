## 六、测试注意事项

1. **名称唯一性**：站点名称在整个系统中必须唯一，创建和更新时需要检查
2. **权限验证**：确保不同角色的用户只能执行其权限范围内的操作
3. **事务处理**：创建、更新、删除操作需要在事务中执行
4. **分页效率**：大数据量下分页查询需要考虑性能优化
5. **索引使用**：确保查询使用了正确的索引（name, status）
6. **时区处理**：created_at 和 updated_at 需要统一时区
