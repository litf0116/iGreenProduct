## 🏗️ 系统架构分析

### 1. igreen-backend（管理后端）

#### 技术栈
- **框架：** Spring Boot 3.2.0
- **语言：** Java 17
- **ORM：** MyBatis Plus
- **数据库：** MySQL
- **认证：** JWT + Spring Security
- **API 文档：** Swagger (Knife4j)
- **构建工具：** Maven
- **测试：** JUnit + H2

#### 目录结构
```
igreen-backend/src/main/java/com/igreen/
├── common/          # 公共组件
│   ├── config/       # 配置类
│   ├── exception/    # 异常处理
│   ├── result/       # 统一响应封装
│   └── utils/        # 工具类
├── domain/          # 领域模型
│   ├── controller/   # REST 控制器
│   ├── entity/       # JPA 实体
│   ├── dto/          # 数据传输对象
│   ├── service/      # 业务逻辑
│   ├── enums/        # 枚举类型
│   └── mapper/       # MyBatis 映射器
└── resources/       # 资源文件
```

#### API 端点

| 控制器 | 端点 | 功能 |
|--------|------|------|
| **AuthController** | `/api/auth` | 登录、注册、令牌刷新 |
| **TicketController** | `/api/tickets` | 工单 CRUD、状态流转 |
| **TemplateController** | `/api/templates` | 模板管理 |
| **SitemapController** | `/api/sitemap` | 站点管理 |
| **ConfigController** | `/api/configs` | 系统配置 |
| **SiteController** | `/api/sites` | 站点管理 |
| **SiteImportExportController** | 站点导入导出 |
| **FileController** | `/api/files` | 文件管理 |

#### 核心功能

**工单管理：**
- ✅ 创建工单
- ✅ 分配给工程师
- ✅ 状态流转（待办→派单→进行中→已完成）
- ✅ 步骤化任务管理
- ✅ 评论系统
- ✅ 步骤更新

**用户和权限：**
- ✅ 用户认证（JWT）
- ✅ 角色管理
- ✅ 权限控制（@PreAuthorize）

**站点管理：**
- ✅ 站点 CRUD
- ✅ 站点导入导出
- ✅ 批量操作

**系统配置：**
- ✅ 系统参数管理
- ✅ 模板管理
- ✅ Sitemap 生成

### 1. 架构优势

#### ✅ 清晰的分层架构
- **后端：** Controller → Service → Mapper → Database
- **前端：** Components → API Client → State Store
- **优点：** 职责分离、易于维护、测试友好

#### ✅ 统一的技术栈
- **三个系统都使用：** React 18 + TypeScript 5
- **优点：** 开发体验一致、组件可复用、维护成本低

#### ✅ 现代化工具链
- **后端：** Spring Boot 3 + Maven + Knife4j
- **前端：** Vite + Playwright + Radix UI
- **优点：** 开发效率高、构建速度快、测试自动化

2. **模板管理前端页面缺失**
